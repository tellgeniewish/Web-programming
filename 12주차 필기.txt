매개변수 없는 생성자 하나 필요

<jsp:useBean id="login" /> //  클래스 객체를 생성
id: 객체 이름
class: 클래스 이름 -> 패키지 이름을 포함한 full name
scope: 반드시 사용할 필요x->선택o
-속성(page->기본값, request, session, application) 중에 하나 사용 가능
-속성관리(setAttribute, getAttribute)
ex) scope="session"
->객체 생성 전 스코프를 훑어서 같은 이름이 있으면 가져다가 사용함! 없으면 새로 만든다

<jsp:setProperty name="login" /> // setter 호출할 때 사용하는 메소드
객체 확보(생성) 먼저 해야 함!
name: 객체 이름(id) // 필수
property: 소속변수 이름 // 필수
value: 매개변수로 들어감
param
value와 param은 함께 쓰지 않음!
ex) private String name; -> void setName(String name) { this.name = name; }
<jsp:setProperty name="test" property="name" value="김성민" /> -> <% test.setName("김성민"); %>
request에 property가 있다는걸 알고 사용함
1. name와 property 속성만 사용
<jsp:setProperty name="test" property="name" />
-> <% test.setName(request.getParameter("name")); %>
2. name와 property와 param 속성만 사용
<jsp:setProperty name="test" property="name" param="username"/>
-> <% test.setName(request.getParameter("username")); %>
-> property와 param이 같은 경우 param 안 써도 됨!

<jsp:getProperty name="login" /> // getter 호출할 때
객체 확보(생성) 먼저 해야 함!
