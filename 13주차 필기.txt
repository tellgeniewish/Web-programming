12장 표현 언어
표현 언어란?
JSP에서 브라우저의 출력은 주로 표현식 태그를 이용
<%= request.getParameter("userid") %>

표현언어를 이용 --> 무언가를 html로 출력한다(변경 및 수정 불가!!!)
${ param.userid }
${ param['userid'] }
${ param["userid"] }

표현언어
-<%  %>인 표현식 대신에 사용하거나
-내장 객체 액션태그 저장도니 자죠를 쉽게 참조하기 위해 만들어진 언어

${ exp }
-$로 시작
-변수가 {  }안에 들어감(산술, 관계, 논리와 같은 기본적인 연산 가능)

자료유형
-정수형, 실수형
-문자열형
-논리(boolean)형
-null값

상수
-논리값	true, false
-정수형	1, -5
-실수형	3.1, 4.5E+4
-문자열	큰따옴표, 작은따옴표 모두 사용 가능
-아무것도 없다=null

예시) ${ product }
1. _jspService()의 지역변수 출력
2. 내장객체 다 뒤져서 같은 이름의 속성을 찾음
--> page, request, session, application 범위에서 등록된 product를 찾아서 반환
--> 만약 찾을 수 없다면 null 반환 --> ${ null } --> html에는 아무것도 출력되지 않음

표현언어 사용 이유
: null값이 절대 html에 안 나온다

표현 언어 연산자
-이항 산술 연산자: 사칙연산(영어 문자를 사용해도 된다)
-이항 관계 연산자: 대소 비교(영어 문자를 사용해도 된다)
-첨자 연산자
-이항 논리 연산자(영어 문자를 사용해도 된다)
-단항 논리 연산자(영어 문자를 사용해도 된다)
-단항 산술 연산자
-empty 연산자
-삼항 조건 연산자
-

표현 언어 연산자
<td> \${1+2} </td> : 문자열로 인식하라
<td> ${1+2} </td> : 3이 출력됨

empty 연산자
-empty 다음의 피연산자 표현식이 null인지를 검사하여 true 또는 false를 반환하는 연산자
-정의되지 않은 표현식 또는 상수 null을 표현 언어로 출력하면 아무것도 출력되지 않음
${ empty exp } : empty 다음이 null이면 true, 아니면 false
-null
-정의되지 않은 객체
-빈 문자열
-길이가 0인 배열

표현언어 내장객체 p.397
-page
-속성: pageScope, requestScope, sessionScope, applicationScope
--> ${ sessionScope.product }
-요청(request) 매개변수
반환값이 하나일 때: <%= request.getParameter("product") %> == ${ param.product }
반환값이 여러개->배열일 때:
<%=	String[] h = request.getParameter("hobby") %>
	out.println(h[0]);
	out.println(h[1]);
%>
==	${ paramValues.hobby[0] } --> jsp코드라서
	${ paramValues.hobby[1] } --> 반복문 못 돌림...
-헤더
-초기화 매개변수
-쿠키

setAttribute()의 두번째 매개변수에 Object 가능
--> toString()으로 출력됨

쿠키 생성을 표현언어로 쓸 수는 없음!
